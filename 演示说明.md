# 紫燕门店收入智能制单 POC — 演示说明

> 面向场景：零售连锁直营门店「多渠道收入确认 + 自动制单」。
> 演示门店：上海虬长店（门店编码 80010129）。
> 核心目标：用真实原始数据，验证「线上审单 + 自动生成会计凭证」可行。

---

## 一、整体逻辑（一句话）

```
门店多渠道收入数据（结算报表 + 平台流水）
   → 清洗归集 → 渠道初判 → 应收↔实收对账
   → 按「计税 + 入账规则」自动制单 → 程序校验 → 输出凭证
```

全程**金额由程序确定性计算、AI 只做理解与判断**（见第七节分工）。

---

## 二、计税规则（价税分离）

门店零售收入为**含税**金额，记账需做价税分离，增值税率 **13%**：

| 项目 | 公式 |
|---|---|
| 不含税收入 | 含税金额 ÷ 1.13 |
| 销项税额 | 含税金额 − 不含税收入（≈ 含税 × 13 ÷ 113）|

**真实示例**（虬长店 2025-01，门店收款小计 = 103,316.08）：
- 不含税收入 = 103,316.08 ÷ 1.13 = **91,430.16**
- 销项税额 = 103,316.08 − 91,430.16 = **11,885.92**

（与真实人工凭证 0101144 一致，差异仅 0.2 元舍入）

---

## 三、入账规则（制单逻辑）

每类业务对应一条规则（带 `rule_id`，凭证可追溯）：

### 1. 收入确认 `R-REV-DIRECT`
按「门店收款小计」全额挂应收，价税分离确认收入：
```
借  应收账款-直营店         = 门店收款小计（含税）   103,316.08
贷  主营业务收入-直营店     = 收款小计 ÷ 1.13          91,430.16
贷  应交税费-销项税额13%    = 差额                     11,885.92
```

### 2. 成本结转 `R-COST-DIRECT`
```
借  主营业务成本-直营店
贷  发出商品
```
（⚠️ 成本金额需存货系统数据，结算报表无；POC 暂用占位并标注待补）

### 3. 折扣净额法 `R-DISCOUNT`
券 / 储值 / 优惠等部分按净额法冲减收入：
```
借  主营业务收入-折扣      = 折扣额 ÷ 1.13
借  应交税费-销项税额13%   = 差额
贷  应收账款-直营店        = 折扣额（含税）
```

### 4. 外卖应收结转 `R-TAKEAWAY`
外卖收入先暂估，拿到平台账单后差异（抽佣）计平台服务费：
```
借  应收账款-外卖平台
贷  应收账款-直营店（暂估）
```

## 五、事项映射（渠道 → 科目）

系统对结算报表里的每个收款渠道，按下表自动归类、判定拟入科目与处理方式：

| 渠道 | 拟入科目 | 处理方式 | 置信度 |
|---|---|---|---|
| POS-微信 / 支付宝 / 银联 / 现金 | 主营业务收入-直营店 | 实收确认 | 高 |
| 外卖-美团 / 饿了么 | 主营业务收入-直营店(暂估) | 暂估 → 拿账单冲回 | 中 |
| 外卖-京东到家 | 主营业务收入-电商 | 实收确认 | 高 |
| 团购券类 | 主营业务收入-直营店 | 净额法 | 中 |
| 券 / 折扣类、折扣调整 | 主营业务收入-折扣 | 净额法冲抵 | 中 / 高 |
| 储值 / 会员 | 预收账款-储值卡 | 需消费明细核销 | 低 |
| 其它支付 | 待分类 | 人工判断 | 低 |

> 置信度「高」→ 规则自动制单；「中/低」→ 标注、转人工复核或交 AI 归因。

## 六、对账逻辑（审单核心）

把**两个独立数据源**交叉核对，差异自动归因：

| 渠道 | 应收（结算报表）| 实收（平台流水）| 差异含义 |
|---|---|---|---|
| 微信 POS | 微信POS应收 | 微信平台应结金额 | 退款 / 跨月 / 手续费 |
| 美团 / 饿了么 | 外卖原价 | 平台商家应收款 | 平台抽佣 / 服务费 |

**真实示例**（虬长店 2025-01 微信）：应收 32,634.02 vs 实收 32,712.01，差异 **77.99（0.24%）**——两个独立来源高度吻合，证明数据可信。

## 七、AI 与程序的分工（关键）

| 环节 | 谁来做 | 说明 |
|---|---|---|
| 金额计算 / 价税分离 / 制单 / 校验 | **Python（确定性）** | 金额 100% 来自原始 Excel，AI 不碰 |
| 规则学习（从历史账归纳记账逻辑）| **DeepSeek** | 读统计结果，语义归纳 |
| 异常归因（差异 / 缺明细）| **DeepSeek** | 性质判断 + 建议，不出金额 |
| 对话式问答 | **DeepSeek** | 基于实时数据解释分析 |

> 大模型：`deepseek-chat`（DeepSeek-V3）。**只做理解 / 判断 / 表达，绝不生成金额、不创造科目。**

---

## 八、控制原则（可信度红线，7 条）

1. 金额 100% 来自原始 Excel，AI 不生成金额
2. 科目必须来自科目表，AI 不创造科目
3. 借贷平衡由程序确定性校验
4. 中低置信 / 多候选 / 大额 / 异常 → 强制人工复核
5. AI 调用全程留痕（输入 / 输出 / 模型版本 / 提示词版本）
6. 每条分录保留 `source_file` / `source_row` / `rule_id`
7. AI 仅产草稿与建议，不替代财务终判
